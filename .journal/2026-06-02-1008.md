# 2026-06-02 10:08 CST

## Core Decision/Topic

Completed the validation E2E audit and cleanup pass for Uniview. The work moved the project from a partially reliable validation foundation to a current baseline with strict Playwright E2E, Cypress visual E2E, package tests, docs build participation, and a package-by-package audit report.

## Options Considered

- Keep Cypress and Playwright as duplicate gates. This would make Cypress match the full Playwright matrix, but it would make the slower visual/debugging flow less pleasant.
- Use Playwright as the strict matrix and Cypress as a developer-experience runner. This keeps Playwright responsible for exhaustive parity and lets Cypress focus on visible step-through debugging.
- Leave deprecated plugin-as-server runtime paths in place as compatibility shims. This preserved old exports but contradicted the current bridge-client architecture and kept misleading examples alive.
- Remove deprecated server paths and document the bridge-client architecture. This removes dead code and makes future work follow the current design.

## Final Decision & Rationale

Playwright is the strict CI-style baseline and now covers 31 scenarios, including React/Solid benchmark full/incremental paths across worker and node-server modes. Cypress remains a visual/debugging E2E runner with 24 scenarios and step highlighting, rather than duplicating every benchmark parity case.

Deprecated `@uniview/react-runtime/ws-server` and first-party `.server.ts` examples were removed because all supported server-side plugin scenarios now use the bridge-client path. Root docs build was re-admitted to `pnpm build` after adding a Webpack compatibility shim for Fumadocs virtual collection imports.

## Key Changes Made

- Fixed a real incremental rendering bug in `MutableTree`: nested mutations now reattach updated child subtrees through every ancestor so the root tree reflects list/text changes.
- Added regression coverage for nested mutation propagation in `@uniview/host-sdk` and keyed list growth mutation emission in `@uniview/react-renderer`.
- Expanded Playwright Svelte-host benchmark coverage to React/Solid, worker/node-server, full/incremental combinations.
- Refreshed the E2E baseline report to 31 passing, 0 failing, 0 skipped, and 0 top-level errors.
- Improved Cypress visual debugging by asserting non-empty highlighted subjects and removing brittle transient `Submitting...` assertions.
- Added meaningful focused tests for Solid renderer, Solid runtime, and TUI renderer, allowing package scripts to drop `--passWithNoTests`.
- Removed deprecated React runtime `ws-server` export, source entry, server examples, related scripts, and unused `ws` dev dependencies.
- Included docs in the root build and switched docs production build to `next build --webpack` with a Fumadocs virtual-module replacement.
- Removed the unused `apps/*` workspace glob and updated root/package AGENTS knowledge so future work points to current bridge-client architecture.
- Fixed the bridge-server test port selection to use OS-assigned ports instead of a single random port, avoiding `EADDRINUSE` flakes.
