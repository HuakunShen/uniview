## Core Decision/Topic

Added a Cypress E2E runner alongside the existing strict Playwright suite to improve interactive debugging and make E2E operations easier to watch, pause, and understand.

## Options Considered

- Keep only Playwright, using `--ui`, `--debug`, and headed runs for interactive debugging.
- Replace Playwright with Cypress for the baseline E2E suite.
- Add Cypress as a parallel developer-experience runner while keeping Playwright as the strict validation baseline.
- Use Node `.mjs` scripts for Cypress orchestration.
- Use TypeScript scripts executed by Bun, matching the user's preference and the repository's existing Bun usage.
- Rely only on Cypress's built-in Command Log time travel/highlight behavior.
- Add custom test-helper highlighting so target elements are visibly outlined while tests run slowly or are stepped.

## Final Decision & Rationale

Playwright remains the strict E2E validation suite. Cypress was added as a parallel runner because its interactive app is better suited for visually exploring specs, selecting tests, pausing, and stepping through commands.

The Cypress runner scripts are TypeScript files executed by Bun. This keeps the workflow consistent with the user's preference to avoid JavaScript scripts where possible and aligns with the existing Bun-based project scripts.

The Cypress suite reuses the same service fixture startup as Playwright instead of duplicating bridge, plugin-client, and host startup logic. That keeps both E2E runners exercising the same environment and reduces drift between validation modes.

Custom element highlighting was added on top of Cypress's built-in Command Log behavior. Cypress documentation confirms that Open Mode supports time-travel snapshots and highlights elements when hovering relevant Command Log entries, but it does not provide a Playwright-style automatic locator outline for every running command. The helper therefore briefly scrolls target elements into view and outlines them. The helper uses `.should(callback)` rather than `.then()` so it does not break Cypress's retryable query/assertion chain.

## Key Changes Made

- Added Cypress configuration with a larger `1280x800` viewport, screenshots on failure, disabled video, and TypeScript E2E spec discovery.
- Added Cypress specs covering React, Svelte, and Vue hosts across worker, main-thread, and node-server runtime modes.
- Added Svelte host Cypress coverage for React and Solid plugins, including Solid's disabled main-thread mode and benchmark smoke behavior.
- Added shared Cypress flow helpers for opening demos, selecting runtimes, running simple and advanced form flows, visual delays, manual pause mode, input filling, and target-element highlighting.
- Added `scripts/e2e-fixtures.ts` to share bridge/plugin/host startup and teardown between Playwright and Cypress.
- Reduced `e2e/global-setup.ts` to call the shared fixture startup.
- Added Bun-powered Cypress scripts:
  - `pnpm test:e2e:cypress` for headless validation.
  - `pnpm cypress:open` for interactive Open Mode with visual delay.
  - `pnpm cypress:open:pause` for manual pause/Next stepping.
- Added Cypress generated-artifact ignores for downloads, screenshots, and videos.
- Installed Cypress as a dev dependency and allowed it in `pnpm.onlyBuiltDependencies`.
